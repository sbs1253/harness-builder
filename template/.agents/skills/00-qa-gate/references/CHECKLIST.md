# QA 검증 상세 체크리스트

> 이 문서는 `00-qa-gate`가 검증 시 참조하는 상세 기준입니다.

## 1. 환각 방지 (Web Grounding)

**통과 조건**: 타겟 에이전트의 Instructions 또는 Harness Rules에 다음 내용이 포함되어야 함:
- "코드 작성 전 공식 문서 웹 검색"
- "사전 학습 지식에 의존하지 말 것"
- 또는 동등한 의미의 지시

**검사 대상**: 모든 `01-*` 이상 에이전트의 `SKILL.md`

## 2. 보안 네거티브 제약

**통과 조건**: 각 에이전트의 Constraints에 다음이 포함:
- API 키 하드코딩 금지 관련 문구
- `.env.local` 사용 강제 관련 문구
- 파괴적 명령어 금지 관련 문구

**검사 대상**: 모든 `01-*` 이상 에이전트의 `SKILL.md`

## 3. SSOT 참조

**통과 조건**: 다음 지시가 포함:
- "작업 전 `docs/PRD.md` 읽기"
- "작업 전 `docs/PLAN.md` 읽기"

## 4. 역할 분리

**통과 조건**: 
- 두 에이전트의 Boundary(수정 가능 범위)가 겹치지 않음
- Role 설명에서 책임 범위가 명확히 구분됨

## 5. PRD 정합성

**통과 조건**:
- `docs/PRD.md`의 핵심 기능 각각에 대해 담당 에이전트가 존재
- 에이전트가 다루는 기능이 PRD에 명시되어 있음

## 6. MCP/Skills 반영

**통과 조건** (해당 시):
- 설치된 MCP가 있다면, 관련 에이전트의 Instructions에 활용 지시가 포함
- 미설치 시: 이 항목은 자동 통과

## 7. Boundary 명시

**통과 조건**:
- 모든 타겟 에이전트에 `## Boundary (작업 범위)` 섹션 존재
- "수정 가능", "읽기 전용", "접근 금지" 3가지 구분이 명확

## 8. 네거티브 트리거

**통과 조건**:
- 모든 타겟 에이전트의 description에 "개입하지 말아야 할 상황" 명시

## 9. 보호구역 제약

**통과 조건**:
- 모든 타겟 에이전트의 Constraints에 `00-*` 보호구역 수정 금지 문구 포함
